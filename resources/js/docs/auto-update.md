# 🔄 Auto-update

Peermitly can **keep itself up to date automatically**. Once a new version is published, the app downloads and installs it in the background — you always run the latest release without ever visiting a download page again.

You install the very first version manually (see [Setup](/guide/setup)). After that, the app takes over: it checks for updates on launch, fetches the new build, and applies it the next time you restart.

> **Pro feature.** Auto-update is part of Peermitly **Pro**. Every other feature is free for everyone — auto-update is the single capability reserved for Pro.

## 🧭 How it works

1. On launch, the app asks Peermitly whether a newer version exists.
2. If one does, it downloads the update in the background while you keep working.
3. The update is applied on the next restart — no reinstall, no manual download.

You'll find the update status and controls in the **app header** inside Peermitly.

## 🆓 Everything else is free

Auto-update is the **only** Pro feature. Every other part of Peermitly — all stacks and runtimes, databases and services, the Debug tool, the Profiler, the Composer graph, the Sidebar Editor and every site type — is available on every plan at no cost.

## ⬇️ Without Pro

Without Pro you can still run the latest Peermitly — you just update **manually**: download the newest version from your dashboard and replace the app in your **Applications** folder, exactly like the first install. Nothing else changes; all features stay available.
